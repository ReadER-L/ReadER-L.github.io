1.GCC编译

1.简介

GCC（GNU Compiler Collection）是GNU推出的功能强大、性能优越的多平台编译器。
❖ GCC是可以在多种平台上编译出可执行程序的编译器集合，集成C、C++、Objective C、Fortran、Java、Fortran和Pascal等多种语言编译器。

2.c编程步骤

步骤：
1）编辑a.c //vi，vim，emacs，gedit编辑器
2）在终端上编译：
gcc a.c //产生默认的a.out可执行文件

或 gcc –o aaa a.c //产生名为aaa的可执行文件
3）在终端上，执行所生成的可执行文件：
./a.out 或./aaa[可执行文件]

实例：

gcc [选项] 源文件 [目标文件]
示例：
gcc hello.c –o hello
结果：产生hello可执行文件

3.GCC编译流程（重点）

GCC的编译流程分为4个步骤，分别为：
（1）预处理：编译器分析处理源代码文件中的各种宏指令，如＃include，＃if等。
（2）编译：GCC首先要检查代码的规范性、是否有语法错误等，在检查无误后，GCC把代码翻译为汇编语言,生成.s文件。
（3）汇编：编译器把编译生成的汇编代码转成二进制目标代码，生成.o文件
（4）链接：编译器把汇编阶段生成的二进制代码、程序中用到的库文件链接起来，生成可执行文件。

4.GCC常用选项

❖-E：预处理.预处理之后的代码将送往标准输出
❖-S：编译为汇编代码，默认扩展名为.s
❖-c：编译为目标代码文件，不链接库，扩展名为.o
❖-o outfile：指定输出文件的文件名,默认为a.out
❖-Wall ：显示附加的警告信息

2.使用make

在Linux环境中，make是一个非常重要的编译命令。

  1. 简化编译时所需要执行的命令。
  2. make是用于自动编译、链接程序的实用工具。
  3. 在编译完之后，修改了某个源代码，make只对修改过的部分进行编译，其他目标文件不会更改；会依照相依性来更新执行文件。

1.makefile文件

​ make工具最主要最基本的功能是通过makefile文件来描述源程序之间的相互关系并自动维护编译工作。makefile的宗旨就是让编译器知道要编译一个文件需要依赖其他的哪些文件。当那些依赖文件有了改变，编译器会自动的发现最终的生成文件已经过时，而重新编译相应的模块。

makefile文件由一组依赖关系和规则构成。

  1. 每个依赖关系由一个目标和一组该目标所需的
    源文件构成。定义了最终应用程序里的每个文
    件与源文件间的关系。
  2. 规则描述了如何通过这些依赖关系文件创建目
    标。
    如 目标文件：依赖的文件
