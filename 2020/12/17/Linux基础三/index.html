Linux基础三

本文最后更新于：2020年12月20日 晚上

Linux复习笔记第三章

1.用户

1.用户类型

超级用户：root拥有计算机系统的最高权限。UID和GID为0。

系统用户：自动创建与系统服务相关的用户。通常这些账号只能被系统守护进程使用。UID和GID为1~499。

普通用户：由超级用户创建，只能操作拥有权限的文件和目录并管理自己启动的进程。UID和GID从500开始。

2.在linux系统中，用户具有下列属性

➢ 用户名：字母数字和符号组成，系统中唯一。
➢ 口令：用于验证用户身份。
➢ 用户ID（UID）：是系统每个用户都拥有的唯一的识别号。Root的UID为0；系统用户的UID在1-499之间；普通用户从500开始。

➢ 组群ID（GID）：每一个用户都属于一个组。分配方式与UID相同。
➢ 用户主目录：用于保存该用户的文件及目录。默认登录后自动进入该目录。
➢ 全称：用户的详细信息，可以空。
➢ 登录shell：用户登录Linux后进入的Shell环境

◼ Passwd文件中每行定义一个用户账号，一行中又划分为多个字段，各字段用“：”隔开。

2.组群

组群是逻辑地组织用户账号集合的方便途径，它允许用户在组内共享文件。
◼ 每一个文件都有一个用户和一个组的属主。
◼ 将具有相同特性的用户划归为一个组群 。
➢系统组群：安装Linux以及部分服务性程序时，系统自动设置的组群 。
➢私人组群：由超级用户新建的组群 。

1.组群相关文件

1./etc/group

2./etc/gshadow

/etc/gshadow文件用于保存加密的组群口令，只有超级用户才能查看/etc/gshadow文件的内容。

3.管理用户的Shell命令

1.useradd命令

格式：useradd [选项] 用户名
功能：新建用户帐号，超级用户命令。
主要选项说明：
-c 全名(comment)
-d 主目录(directory)
-e 有效期限(expire)
-g 组群ID|组群名 指定用户所属的主要群组
-u 用户ID（user）
-s 登录Shell
如果没有选项，将按系统默认值设置新用户信息。

2.passwd命令

格式：passwd [选项] [用户]
功能：设置或修改用户的口令以及口令的属性。没有加用户名表示修改当前用户。
主要选项说明：
-d 用户名（delete） //删除用户口令
-l 用户名（lock） //锁定用户账号
-u 用户名（unlock） //解除用户账号
-S 用户名（status） //显示账号的状态

3.usermod命令

功能：修改用户的属性，超级用户命令。
格式：usermod [选项] 用户名
主要选项说明：
-c 全名(comment)
-d 主目录(directory)
-g 组群ID或组群名 //指定用户所属的主要群组
-u 用户ID（user）
-l 用户名

4.userdel命令

格式：userdel [-r] 用户名
功能：删除指定用户帐号，超级用户命令。
选项说明 ：
-r(remove):系统不仅删除此用户帐号，还将删除用户
的主目录

5.id命令

格式：id/groups [选项] [用户名]
功能：查看用户的UID、GID和用户所属组群的信息。
没有选项，则显示全部信息。如果不指定用户，则显示当前用户的信息。

6.groupadd命令

功能：新建组群，超级用户命令。
格式：groupadd [选项] 组群名
主要选项说明：
-g 组群ID //指定GID
◼ 该组ID值必须是惟一的，且数值不可为负。
◼ 预设的最小值不得小于500。
◼ 只有root能用该命令

7.groupmod命令

格式：groupmod [选项] 组群名
功能：修改指定组群属性，超级用户命令。
主要选项说明：
-g 组群ID //指定GID
-n 组群名（name） //指定组群名

8.groupdel命令

格式：groupdel 组群名
功能：删除指定的组群，超级用户命令。
注：若该群是一些用户的主要组群，则需要首先删除这些用户后，才能删除这个组群。

9.gpasswd命令

格式：gpasswd [选项] 组群
功能：
gpasswd 组名：修改组密码
gpasswd –r 组名：删除组密码
gpasswd –a 用户名 组名：将某个用户加入组
gpasswd –d 用户名 组名：从组里删除用户
