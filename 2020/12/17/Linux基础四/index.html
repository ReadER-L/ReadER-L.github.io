Linux复习第四章

1.Linux文件系统概述

◆Linux采用虚拟文件系统技术，可支持多种常见
的文件系统，并允许用户在不同的磁盘分区上安
装不同的文件系统。
◆Linux支持的文件系统类型主要有：
➢MSDOS：MS DOS采用的FAT文件系统。
➢VFAT：Windows中通用的文件系统。
➢NFS：网络文件系统（Network File System）。
➢ISO9660：CD-ROM的标准文件系统。
➢XFS：SGI开发的日志文件系统

补充：

➢ 簇是一组连续的扇区；一般是2n(n为整数)个盘块。
➢簇的最大值为64个扇区，即32KB
➢一个簇应包含扇区的数量与磁盘容量的大小
直接有关，当以它为单位分配并链接，可减
少访问时间，但增大了内部碎片。

硬盘类型：IDE接口硬盘和SCSI接口硬盘

◆一台计算机上一般有两个IDE接口，每个IDE口上可连接两个硬盘设备（主盘和从盘）。
➢IDE0接口上的主盘称为/dev/hda
➢IDE0接口上的从盘称为/dev/hdb
◆SCSI设备：
第一块SCSI设备称为：/dev/sda
第二块SCSI设备称为：/dev/sdb

1.EXT（Extended File System）

➢文件系统系列（EXT2、EXT3、EXT4）是专为Linux设计的文件系统。
➢高实用性
➢数据的完整性
➢更快的速度
➢方便的数据转换

2.SWAP文件系统

➢用于Linux的交换分区
➢提供虚拟内存服务
➢一般在Linux系统安装过程中自动完成
➢Linux不支持使用mount命令挂载SWAP文件系统

3.文件系统的挂载与卸载

◆挂载就是将存储介质的内容映射到指定的目录中，此目录即为该设备的挂载点。
➢硬盘上的各个磁盘分区都会在启动过程中自动挂载到指定的目录，并在关机时自动卸载。
➢移动存储介质既可以在启动时自动挂载，也可以在需要时手动挂载/卸载。
◆Linux 为用户的后续挂载专门提供了/mnt 或/media目录作挂载点。只有root有权执行挂载与卸载操作。

4.配额的基本概念

➢磁盘配额是一种磁盘空间的管理机制。
➢限制用户或组群在某个特定文件系统中所能使用的最大空间。分用户级和组群级的配额管理。
➢配额管理文件保存于实施配额管理的那个文件系统的挂载目录中
➢aquota.user文件保存用户级配额的内容
➢aquota.group文件保留组群级配额的内容。

◆据配额特性将配额分为硬配额和软配额
➢硬配额是用户和组群可使用的最大值。
➢软配额是用户和组群可使用空间，但系统允许软配额在一段时间（过渡期）内被超过，过渡期后，用户不能写入更多的文件。
➢通常硬配额大于软配额。
◆只有采用Linux文件系统ext的文件系统（磁盘分区）才能进行配额管理。

步骤：

◆修改/etc/fstab文件，设置某个文件系统的配额为usrquota或是grpquota
◆重启系统
◆用quotacheck命令创建配额文件
◆用edquota 命令编辑配额文件：edquota 用户
◆用quotaon命令启动配额文件
◆用quotaoff命令关闭配额文件

1.quotacheck命令

格式： quotacheck 选项
功能： 检查文件系统的配额限制，创建配额管理文件
选项：

-a（all）：扫描/etc/mtab文件中所有挂载的文件系统
-g（group）：检查每个组群占用的目录和文件
数目，并创建aquota.group文件
-u（user）：检查每个用户占用的目录和文件
数目，并创建aquota.user文件
