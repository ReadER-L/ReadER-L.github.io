18.sqlmap设置HTTP协议私钥

当服务器需要适当的客户端证书和用于身份验证的私钥时，应该要运用此项。提供的值应该是一个PEM格式的KEY_FILE,其中包含证书和私钥

–auth-file 文件名

19.sqlmap设置安全模式

避免在多次请求后销毁会话,因为在探测过程中肯定会有一定数量不成功的请求，在这种情况下，web应用有可能会销毁会话

–safe-url/–safe-post/–safe-req（用于-r的）/–safe-freq（用于-l ）

20.sqlmap设置忽略url编码

据参数的位置（例如GET），在默认情况下它的值可以使url编码的。在某些情况下，后端的web服务起不遵循RFC标准，需要以原始的非编码形式发送值，在这种情况下使用–skip-rulencode 不进行url加密

3.sql性能优化

0.-o参数可以开启所有性能优化参数

1.sqlmap设置持久HTTP连接，让我们不用重新建立连接，加快探测，保护了目标的资源

在HTTP报文中设置connection: keep-alive

2.sqlmap设置不接受HTTP body

在sqlmap中设置空连接，不接受响应包中的body，只接受响应头，加快我们的请求速度

常用在盲注过程中

–null-connection：为了检索没有body响应的内容

3.sqlmap设置多线程加快速度

sqlmap设置同时发送多少个HTTP请求的多线程

1.sqlmap强制设置dbms

默认情况下sqlmap会自动识别探测目标web应用程序的后端数据库管理系统(dbms)，以下列出sqlmap完全支持的dbms种类。

MYSQL、oracle、Microsoft sql server、IBM db2、sqlite、firebird、sybase、sap maxdb、hsqldb、informix

–dbms 数据库管理系统名称[版本号]

例如：–dbms mysql 5.0、–dbms microsoft sql server 05

2.sqlmap强制指定os

默认情况下sqlmap会自动识别探测目标web应用程序后端操作系统(os)，以下列出sqlmap完全支持的os种类

Linux、Windows

例如：–os windows 或者–os linux
