python学习之异常处理

本文最后更新于：2020年6月17日 上午

python学习之异常处理

常见的异常：

2.AssertionError（断言语句assert失败）

2.AttributeError（尝试访问未知的对象属性）

3.indexError：索引超出序列的范围

4.keyError：字典中查找一个不存在的关键字

5.NameError：尝试访问一个不存在的变量

6.OSError：操作系统产生的异常

7.SyntaxError：Python的语法错误

8.TypeError ：不同类型间的无效操作

9.ZeroDivisionError：除数为零

10.try-except语句（异常捕获机制）

用于检测和处理异常（注意：一旦try语句检测范围内出现异常的话，剩下的语句都不会执行）

出于程序员的角度，我们有时候需要知道倒是异常的原因。因此我们可以使用as把具体的错误信息打印出来

当然，我们可以针对不同异常设置多个except

我们还可以对多个异常进行统一的处理

11.try-finally语句（实现就算出现异常了，也可以执行一些不得不执行的收尾工作）

比如当有一个文档确实存在。open()函数正常返回文件对象。但异常却发生了在成功打开文件后的sum=1+’1’语句上了。此时Python将直接跳到except语句，也就是说文件打开了，但是没有执行关闭的命令

因此我们可以使用try-finally来完成收尾工作

12.raise语句（自己抛出一个异常）

13.丰富的else语句（else还可以和for语句和while语句构成‘干完了能怎么样，干不完就别想怎样’的句式）

说白了，else语句块只能在循环完成后执行，也就是说如果循环中间使用break跳出循环，那么else里面的内容就不会被执行。

else还可以和try语句块进行搭配，没有问题那就干吧

14.简洁的with语句

有时候可能觉着打开文件又要关闭文件，还要关注异常处理有点麻烦。所以Python提供了一个with语句。利用这个语句抽象出文件操作中频繁使用的try/except/finally相关细节。
